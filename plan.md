# 合約簽核系統 產品規劃

## 系統概述

提供企業內部合約全生命週期管理，涵蓋合約建立、多層級簽核流程、狀態追蹤與歷史查閱，讓簽核人員與申請人皆能即時掌握合約進度。

---

## 功能架構（兩層式）

### 1. 合約管理

#### 1.1 建立合約
- 填寫合約基本資訊（標題、對象、金額、有效期限、合約類型）
- 上傳合約文件（PDF / Word，支援多附件）
- 選擇簽核流程範本或自訂簽核人員與順序
- 儲存草稿或直接送出簽核

#### 1.2 合約列表
- 依狀態篩選：草稿、簽核中、已完成、已退回、已作廢
- 依關鍵字、日期區間、合約類型、金額範圍搜尋
- 列表欄位：合約編號、標題、申請人、目前簽核人、狀態、更新時間
- 支援排序與分頁

#### 1.3 合約詳情
- 顯示合約完整資訊與附件預覽
- 顯示當前簽核節點與整體簽核進度條
- 顯示每個節點的簽核人、操作（同意／退回）、時間與意見
- 提供下載合約文件與簽核紀錄 PDF 功能

#### 1.4 合約編輯與作廢
- 草稿狀態可編輯所有欄位
- 簽核退回後可修改後重新送出
- 授權人員可將合約標記為作廢並填寫原因

---

### 2. 簽核流程

#### 2.1 待我簽核
- 列出所有輪到本人簽核的合約
- 顯示優先程度（依截止日期或金額標示）
- 提供快速同意／退回操作，可填寫簽核意見
- 支援批次同意多筆合約

#### 2.2 我的簽核中合約
- 列出由本人發起、目前仍在流程中的合約
- 顯示目前卡在哪個節點、等待哪位人員簽核
- 提供催簽功能（發送提醒通知給待簽人）
- 可查看每個節點的預計完成時間

#### 2.3 我的已完成合約
- 列出本人發起且流程已全數通過的合約
- 提供下載已簽核合約文件功能
- 支援依時間、類型、對象篩選查詢
- 顯示完成日期與最終簽核人

#### 2.4 我的退回合約
- 列出被任一節點退回的合約
- 顯示退回節點、退回人員與退回原因
- 提供修改後重新送出的操作入口

---

### 3. 簽核流程設定

#### 3.1 流程範本管理
- 建立常用簽核流程範本（如：採購合約、NDA、服務合約）
- 設定流程節點：依序簽核或會簽（同時送出）
- 設定每個節點的簽核人（固定人員或依職位動態指定）
- 啟用／停用範本

#### 3.2 簽核規則設定
- 依合約金額自動套用不同簽核層級
- 設定逾期未簽自動升級（Escalation）規則
- 設定代理簽核人（當主要簽核人不在時）

---

### 4. 通知與提醒

#### 4.1 系統通知
- 合約送出時通知第一關簽核人
- 每個節點通過後通知下一關簽核人
- 合約全數通過或退回時通知申請人
- 系統內通知中心（鈴鐺圖示）顯示未讀通知

#### 4.2 Email / 訊息推播
- 可設定個人接收 Email 通知的觸發條件
- 提供催簽通知郵件範本
- 支援整合企業通訊工具（如 Teams、Slack）推播通知

---

### 5. 儀表板與統計

#### 5.1 個人儀表板
- 快速數字：待我簽核數、我的簽核中數、本月已完成數
- 待簽合約到期倒數提醒
- 最近活動紀錄（最近 10 筆操作）

#### 5.2 管理報表（管理員）
- 各部門合約數量與狀態分布圖
- 平均簽核時長趨勢圖
- 逾期未完成合約清單
- 支援匯出 Excel / CSV

---

### 6. 權限與帳號管理

#### 6.1 角色與權限
- 預設角色：一般使用者、部門主管、系統管理員
- 可自訂角色並設定可存取的功能模組
- 合約可見範圍控制（僅本人、部門、全公司）

#### 6.2 使用者管理（管理員）
- 新增／停用使用者帳號
- 指定部門與角色
- 設定代理人期間（請假期間自動轉派）

---

## 核心使用流程

```
申請人建立合約 → 選擇簽核流程 → 送出
  → 系統通知第一關簽核人
  → 簽核人同意 → 進入下一節點（循環）
  → 簽核人退回 → 通知申請人修改重送
  → 所有節點通過 → 合約完成，通知申請人
```

---

## 技術備註

- 合約文件儲存建議使用物件儲存（如 S3），不存入資料庫
- 簽核流程引擎需支援：循序、會簽、條件分支三種模式
- 所有簽核操作需留存不可竄改的稽核日誌
- 前端建議 SPA 架構，後端提供 RESTful API
